<objective>
## Goal
Wire SMAPI events to capture all income/expenses, classify items by category, calculate profitability, track legendary fish, and generate season summaries.

## Purpose
This is the core engine — without it, there's no data to display. Completes Phase 1.

## Output
- TrackingService: captures shipping bin contents, shop purchases via SMAPI events
- CategoryClassifier: maps items to Farming/Foraging/Fishing/Mining/Other
- ProfitabilityCalculator: revenue - seed cost - fertilizer cost per item
- SeasonSummaryService: aggregates daily data, finds best day
- Legendary fish detection
- All data persisted to SQLite via Repository
</objective>

<acceptance_criteria>

## AC-1: Daily income tracked by category
```gherkin
Given a player ships items at end of day
When DayEnding event fires
Then a DailyRecord is created with income split by Farming/Foraging/Fishing/Mining/Other
```

## AC-2: Item transactions recorded
```gherkin
Given items are shipped
When the tracking service processes the shipping bin
Then each item is recorded as an ItemTransaction with name, category, quantity, price, and cost basis
```

## AC-3: Legendary fish detected
```gherkin
Given a player ships a legendary fish
When the fish record is created
Then IsLegendary is set to true
```

## AC-4: Season summaries generated
```gherkin
Given a season ends (day 28)
When DayEnding fires on the last day
Then a SeasonSummary is created with totals per category and best day identified
```

## AC-5: Expenses tracked
```gherkin
Given a player buys items from shops
When inventory changes after shop interaction
Then expenses are recorded in the daily record
```

<task type="auto">
  <name>Task 1: CategoryClassifier + ProfitabilityCalculator</name>
  <files>LaCompta/Services/CategoryClassifier.cs, LaCompta/Services/ProfitabilityCalculator.cs</files>
  <action>
    CategoryClassifier:
    - Use StardewValley.Object.Category constants to map items
    - Category -75 = Vegetables (Farming), -79 = Fruit (Farming), -80 = Flowers (Farming)
    - Category -81 = Forage (Foraging), -23 = Bait/Tackle (Fishing), -4 = Fish (Fishing)
    - Category -15 = Metal (Mining), -12 = Minerals (Mining), -2 = Gem (Mining)
    - Everything else = Other
    - Also check item.ParentSheetIndex for edge cases
    - Return string: "Farming", "Foraging", "Fishing", "Mining", "Other"

    ProfitabilityCalculator:
    - Get item sell price from item.Price or item.sellToStorePrice()
    - Look up seed cost from Game1.objectData or crop data
    - Estimate fertilizer cost if applicable
    - Return CostBasis (seed + fertilizer) and Profit (revenue - cost)
    - For non-crop items (fish, forage, minerals), CostBasis = 0

    Use parallel agents for these two independent services.
  </action>
  <verify>dotnet build succeeds</verify>
  <done>Supports AC-1, AC-2</done>
</task>
