<task type="auto">
  <name>Task 1: Create build and release workflows</name>
  <files>.github/workflows/build.yml, .github/workflows/release.yml</files>
  <action>
    1. Create .github/workflows/build.yml:
       - Trigger: push to any branch, pull_request to main
       - Runner: ubuntu-latest
       - Steps: checkout, setup-dotnet (6.0.x), dotnet restore, dotnet build
       - SMAPI dependency: download and extract SMAPI for the build to resolve game refs
         OR use the GamePath trick with a stub (ModBuildConfig can build without the game)

    2. Create .github/workflows/release.yml:
       - Trigger: push tags matching 'v*'
       - Runner: ubuntu-latest
       - Steps: checkout, setup-dotnet (6.0.x), dotnet restore, dotnet build
       - Package: collect mod files (DLL, manifest.json, Assets/, dependencies) into LaCompta/ folder
       - Zip: create LaCompta-{version}.zip
       - Release: use softprops/action-gh-release to create GitHub Release with the zip

    Note on SMAPI build without game:
    - Set SMAPI_MODS_PATH and GamePath environment variables
    - ModBuildConfig can build without the actual game installed if you provide stubs
    - OR use the community approach: download SMAPI + game stubs for CI

    Avoid:
    - Requiring Stardew Valley to be installed on CI
    - Hardcoded paths
    - Publishing debug builds (use Release configuration)
  </action>
  <verify>Workflow YAML is valid (can check with actionlint if available)</verify>
  <done>AC-1, AC-2 satisfied</done>
</task>
