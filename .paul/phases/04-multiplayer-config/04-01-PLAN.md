<objective>
## Goal
Add split-screen multiplayer support with per-player data isolation, and integrate GMCM (Generic Mod Config Menu) for in-game configuration with a dashboard button.

## Purpose
Multiplayer players each need their own tracked data. GMCM gives users a familiar in-game settings UI without needing the console.

## Output
- PerScreen<T> wrappers for per-player state in split-screen
- ModConfig model with SMAPI config persistence
- GMCM integration (optional soft dependency)
- Player selector in dashboard API for multiplayer views
- Updated documentation
</objective>

<acceptance_criteria>

## AC-1: Split-screen multiplayer isolates per-player data
```gherkin
Given two players in split-screen mode
When each player ships different items
Then each player's daily record only contains their own items
And the dashboard shows data filtered by the current screen's player
```

## AC-2: ModConfig persists settings via SMAPI config
```gherkin
Given the mod is installed
When the player changes settings (port, auto-open dashboard)
Then settings persist in config.json via SMAPI's config API
And settings are loaded on next game launch
```

## AC-3: GMCM menu registers when GMCM is installed
```gherkin
Given GMCM is installed
When the player opens the mod config menu
Then LaCompta settings appear with toggles for auto-open dashboard and port
And a paragraph instructs how to open the dashboard
```

## AC-4: Mod works without GMCM
```gherkin
Given GMCM is NOT installed
When the mod loads
Then no error occurs
And settings can still be edited via config.json manually
```

## AC-5: Dashboard API supports player filtering
```gherkin
Given multiple players have tracked data
When accessing /api/seasons?playerId=X
Then only that player's data is returned
And the default (no playerId) returns the current player's data
```

       - WebServerPort (int, default 5555)

    2. Create LaCompta/Integrations/IGenericModConfigMenuApi.cs:
       - Copy the GMCM API interface (Register, AddBoolOption, AddNumberOption, AddParagraph, AddSectionTitle)
       - This is the standard pattern — no NuGet package needed

    3. Update ModEntry.cs:
       - Add ModConfig field, load in Entry() via Helper.ReadConfig<ModConfig>()
       - Wrap _tracker, _seasonSummary in PerScreen<T> for split-screen support
       - Use Config.WebServerPort when starting the web server
       - If Config.AutoOpenDashboard, auto-open browser on SaveLoaded
       - Register GameLaunched event for GMCM setup
       - In OnGameLaunched: get GMCM API, if available register:
         - Section: "Dashboard"
         - Bool: "Auto-open dashboard on save load"
         - Number: "Web server port" (min 1024, max 65535)
         - Paragraph: "Run lacompta_open in SMAPI console to open the dashboard"
       - Update OnDayStarted/OnDayEnding to use PerScreen values

    Avoid:
    - Making GMCM a hard dependency
    - Breaking single-player mode
    - Changing the web server to run per-screen (single server is fine, filter by playerId)
