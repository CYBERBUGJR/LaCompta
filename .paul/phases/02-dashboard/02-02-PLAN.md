- Chart.js charts: line (daily trends), bar (category comparison), pie (income breakdown)
- Dynamic pie charts / histograms (clickable to sort)
- Season selector navigation
- Summary cards with key stats
- Legendary fish hall of fame
- Top profitable items table
- Capitalist comedy Easter eggs
- All data fetched from existing REST API endpoints

    The dashboard is a single HTML page with embedded CSS and JS (or separate files served by the web server).
    All data comes from the existing REST API via fetch() calls.

    Requirements for /frontend-design:
    - Stardew Valley pixel art aesthetic: warm earth tones (#1a1a2e background, #e6d9a8 text, browns, greens)
    - Pixel-art-style borders (double-line or game-like panel borders)
    - Monospace or pixel font (e.g., Press Start 2P from Google Fonts, or system monospace)
    - Color-coded categories: Farming=#4CAF50 green, Foraging=#8BC34A lime, Fishing=#2196F3 blue, Mining=#FF9800 orange, Other=#9E9E9E gray
    - Chart.js via CDN for all charts (no npm/build step)
    - Layout: header with summary cards → season tabs → charts row → profitability table → legendary fish section
    - Responsive (works on different screen sizes)
    - Comedy mood: funny headers, tooltips, empty states ("No data yet... your farm is a financial black hole")
    - Dynamic: season selector fetches data and re-renders charts
    - Reference spreadsheet layout: daily rows per season, category columns, totals, color coding

    Update ApiController.cs:
    - Change ServeHomePage to read and serve the HTML file from embedded resources or from disk
    - Add routes for /style.css and /app.js if using separate files
    - Or serve everything as a single HTML page with inline CSS/JS

    Update LaCompta.csproj:
    - Add Content/EmbeddedResource for the HTML/CSS/JS assets

    Avoid:
    - External build tools (webpack, npm) — everything must be static files
    - Heavy frameworks (React, Vue) — vanilla JS only
    - External dependencies beyond Chart.js CDN
