<objective>
## Goal
Create an embedded HTTP web server inside the mod that serves JSON REST API endpoints for all tracked statistics.

## Purpose
The API layer is the foundation for the dashboard UI (Plan 02-02). It also enables future integrations (Prometheus, external tools).

## Output
- Background HttpListener web server (starts on SaveLoaded, stops on ReturnedToTitle)
- REST API endpoints returning JSON for all data queries
- ModEntry wired to start/stop server lifecycle
- SMAPI console command to open dashboard in browser
</objective>

<task type="auto">
  <name>Task 1: WebServer + ApiController + wire ModEntry</name>
  <files>LaCompta/Web/WebServer.cs, LaCompta/Web/ApiController.cs, LaCompta/ModEntry.cs, LaCompta/Data/Repository.cs</files>
  <action>
    WebServer.cs:
    - HttpListener on configurable port (default 5555)
    - Start/Stop methods, runs on background thread
    - Request routing: parse URL path, delegate to ApiController
    - CORS headers for local browser access
    - Graceful shutdown on Stop()
    - Error handling: log exceptions but don't crash

    ApiController.cs:
    - Constructor takes Repository + IMonitor
    - Routes:
      GET /api/daily?season=X&year=N&playerId=X → DailyRecord[]
      GET /api/seasons?playerId=X → SeasonSummary[]
      GET /api/profitability?season=X&year=N&limit=10&playerId=X → ItemTransaction[]
      GET /api/fish/legendary?playerId=X → FishRecord[]
      GET /api/summary → { totalSeasons, totalIncome, totalExpenses, legendaryFishCount }
      GET / → serve dashboard HTML (placeholder for now, just "LaCompta Dashboard - Coming Soon")
    - All responses JSON with Content-Type application/json
    - Use System.Text.Json for serialization
    - Query string parsing for parameters

    ModEntry.cs updates:
    - Create WebServer on SaveLoaded, start it
    - Stop WebServer on ReturnedToTitle event
    - Add "lacompta_open" console command to open browser via System.Diagnostics.Process.Start

    Repository.cs additions (if needed):
    - GetOverallSummary() method for /api/summary endpoint

    Avoid:
    - Kestrel or ASP.NET (too heavy for a game mod)
    - Blocking the game thread (all HTTP on background thread)
    - Port conflicts: catch AddressInUseException, log warning
  </action>
